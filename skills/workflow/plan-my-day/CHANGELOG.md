# Changelog

## 1.10.0

- Suppress already-actioned review asks. Phase 3 gains a "Resolve review state for referenced PRs" sub-step: after Slack dedup, fenced message bodies are scanned for PR references (`github.com/<owner>/<repo>/pull/<N>` URLs, plus `#N` only with repo context), mapped to `(repo, N, message_ts)`, and review state for those PRs plus all `reviewRequested` PRs is fetched in one `gh api graphql` batch (`state`, `headRefOid`, `reviews(last: 20) { author { login }, state, submittedAt, commit { oid } }`, `viewer { login }` → `ME_LOGIN`) — no new `allowedTools` entry needed. Phase 4 gains a state-aware suppression rule keyed on whether my latest review covers the PR's current head, not on ask timing: a "Review PR #N" bullet is dropped when the PR is merged/closed, or when my latest review is `APPROVED` **and** GitHub is not actively re-requesting my review (`reviewRequested_repoI` records are tagged `isReviewRequested` and take precedence over the approval drop, so a fresh re-request after an approval still renders); kept in "Do first" when my latest review is `CHANGES_REQUESTED` and the head SHA moved since it (author pushed); moved to a quiet "Already handled" sub-note when my review's `commit.oid` matches `headRefOid` or was only `COMMENTED`; anything uncertain (no review, lookup error) stays in "Do first". Freshness is decided by commit-SHA equality (`review.commit.oid` vs `headRefOid`), not timestamps, so a preserved-date rebase can't make a stale PR read as handled. `message_ts` is now cosmetic (note wording only), so multiple Slack mentions of the same PR no longer risk suppressing the wrong bullet. Trust boundaries table gains a row for review author logins, states + commit SHAs (structured, compared not quoted, LOW).
- CodeRabbit review fixes: `date` calls used for day-plan windows now fall back to GNU `date -d` when BSD `date -j`/`-v` isn't available, so Phase 1/setup works on macOS and Linux; the tracker-search JQL drops the rolling `-7d` window in favour of `statusCategory != Done` so older untouched tickets stay visible; the github ticket-extraction rule prefers the first 3+ digit run when multiple numbers appear (matching `references/tracker.md`); the done-label match is now an exact (trimmed, case-insensitive) match instead of substring containment, so `Review PR #1` can no longer flip `Review PR #10`; `close` now queries issues with `--state all` (not just `open`) and refuses cleanly with a "already closed" message instead of re-running C1–C4 against an already-closed issue; issue create/edit now write the body to a temp file and pass `--body-file` instead of inlining `--body`; the monthly-review previous-month date fallback anchors to the first of the current month before subtracting so it no longer overflows on 31-day months.

## 1.9.0

- Fence external content per `references/prompt-injection-defense.md`. monthly-review M2 fences the previous-month retro issue body, runs the canonical injection-keyword scan, drops only matched bullets (not the whole hint or section), emits a one-line warning, and hardens the posture-hint synthesis prompt (one sentence, paraphrase, no quotes or URLs). standup S0 and close-day C0 fence `ISSUE_BODY` at the read site so downstream phases reuse the same fenced view; standup S4 only echoes the freshly computed Standup block, never re-relays other sections. close-day C2 switches tick-matching from free-form LLM extraction to `TICKET_ID_REGEX` over Done bullets so tampered Done entries cannot steer checkbox flips beyond their literal token content. Trust Boundaries section added to `SKILL.md`.
